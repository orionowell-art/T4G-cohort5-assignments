When you type a website address like google.com into your web browser and press enter, the browser first checks to find the website's IP address by contacting the domain name system. Once it gets the IP address, it connects to the web server where the website is hosted using the internet. The browser then sends a request asking for the website's content. The load balancer helps distribute traffic amongst different servers. The web server processes the request and sends back the website's files such as HTML,CSS,Javascript. Finally the browser reads these files and displays the web page on the screen so that you can interact.
A client is a device,application or program that requests services or resources while a server is a device or program that provides those services or responses to the request. The client is the requester while the server is the provider. For example when you use google chrome to open a website, chrome is the client because it asks for the page and the web server that stores and sends the website data is the server.
DNS which is Domain Name System is like the internet's phonebook. It takes a website name that people can easily remember such as google.com and finds the IP address that computers use to locate that website. We need DNS because remembering website names is much easier than memorizing the IP address.  
A firewall acts like a security guard for the website or network. It checks the traffic coming and going in and out , allowing safe connections  while blocking anythung that looks harmful or suspicious. A load balancer on the other hand is like a traffic controller.It distributes the traffic amongst different servers. The difference between the two is that a firewall is mainly concerned with keeping the website secure while a load balancer is responsible for keeping the website running smoothly and handling lots of visitors efficiently.
A web server is responsible for receiving requests from a web browser and delivering website contents such as HTML,CSS and Javascript files.An application server handles the logic behind a website or a web application.A backend developer works more with the application server because that is where most of the application's functionality is built.
The client does not communicate directly with the database because it would expose important data and make the system less secured.Instead, requests go through the application server, which checks and processes them before accessing the database. The application server improves security, validates requests and ensure users only access the data they are authorized to see. It also helps the system to run more efficiently.
A university department page such as ug.edu.gh/statistics is more likely to be a static website because it mainly displays information that does not chane often such as courses,staff and contact details. An MTN Momo account page is a dynamic website because it displays personalized information like your account balance and transaction history.The content changes based on the user who logs in.
Frontend development focuses on what users and interact with, such as layout,colors and the fonts.Backend development focuses on the server,database and application logic that processes requests and manages data behind the scenes.For a login feature, the backend is responsible for checking the password against the database.This is because the database is stored on the server and keeping password verification on the backend protects sensitive information. The frontend only collects the username and password from the user and sends them securely to the backend for verification.
